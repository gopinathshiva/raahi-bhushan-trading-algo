    Bidirectional Sync

    Sync all event data between SQLite (local) and Supabase (cloud) for a date range.
    Rows present in one database but missing in the other are pushed across — no row is deleted.

    Tables synced: snapshots, position_changes, notifications, ai_chat_history. Reference tables (master_contract, subscriptions, etc.) are not included. Profiles have a dedicated sync card below.

    Sync Profiles

    One-way sync: Local SQLite → Supabase. Profiles are matched by slug. Missing profiles are inserted (preserving local id); existing profiles with field differences are updated to match local. Profiles are never deleted from Supabase — slugs present only in Supabase are listed as informational and left untouched. To remove a profile from Supabase, use Manage Profiles. After insert, the Supabase profiles_id_seq is bumped to prevent future id collisions.
